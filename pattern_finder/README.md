# 形态相似检索模块

基于历史形态相似度检索，找到与当前股票技术形态最相近的历史成功案例，用于辅助选股决策。

## 核心理念

不是让 AI 直接选股，而是构建"历史形态相似检索 + 后续收益验证"系统：

1. **结构化成功案例** - 定义什么是成功（涨幅 + 回撤控制）
2. **特征化图形** - 将 K 线图转换为可比较的数值特征
3. **相似性匹配** - 找到历史上"长得像"的案例
4. **后验统计** - 统计这些案例的后续表现（胜率/盈亏比/收益分布）

**输出的不是"买入信号"，而是"历史相似样本的后验分布"**，供人工决策参考。

## 快速开始

```bash
cd pattern_finder

# 1. 构建样本库（茅台、平安、五粮液，2020-2024）
./quickstart.sh build 600519,000001,000858 --start 20200101 --end 20241231

# 2. 查询单只股票的相似案例
./quickstart.sh query 600519

# 3. 扫描当前持仓的形态信号
./quickstart.sh scan

# 4. 演示模式（使用模拟数据）
./quickstart.sh demo
```

## 工作流程

```
┌─────────────┐
│ 1. 数据加载 │ → 使用 shared.data_provider（baostock/FutuOpenD/Eastmoney）
└─────────────┘
       ↓
┌─────────────┐
│ 2. 特征提取 │ → 技术指标：MA/MACD/RSI/KDJ/布林带/ATR/OBV
└─────────────┘   归一化：价格序列、成交量序列
       ↓
┌─────────────┐
│ 3. 滑动窗口 │ → 60日观察窗口 + 20日未来表现
└─────────────┘   步长5日，生成历史样本
       ↓
┌─────────────┐
│ 4. 成功标注 │ → 成功案例：20日内涨幅≥10% 且最大回撤≤8%
└─────────────┘
       ↓
┌─────────────┐
│ 5. 相似检索 │ → 混合检索：余弦相似度（快速召回）+ DTW（精细排序）
└─────────────┘   召回 top 20 相似案例
       ↓
┌─────────────┐
│ 6. 后验统计 │ → 胜率、盈亏比、收益分布、分年度表现
└─────────────┘
       ↓
┌─────────────┐
│ 7. 报告生成 │ → HTML 报告：K线图、指标、相似案例对比、统计分析
└─────────────┘
```

## 配置参数

### 窗口参数

```python
LOOKBACK_DAYS = 60        # 特征观察窗口（过去多少个交易日）
FORWARD_DAYS = 20         # 未来预测窗口（评估后续表现）
TOP_K_SIMILAR = 20        # 召回相似案例数量
```

### 成功案例定义

```python
SUCCESS_RETURN_THRESHOLD = 0.10     # 未来20日内涨幅阈值（10%）
SUCCESS_DRAWDOWN_LIMIT   = 0.08     # 期间最大回撤上限（8%）
```

**为什么这样定义？**
- 10% 涨幅：过滤掉平庸案例，只保留明显上涨的形态
- 8% 回撤：控制风险，避免"先大跌再反弹"的假阳性案例
- 保守标准：宁可漏掉一些机会，也要确保样本质量

### 技术指标参数

```python
MA_PERIODS    = [5, 10, 20, 60]     # 均线周期
MACD_PARAMS   = (12, 26, 9)         # MACD 快线/慢线/信号线
RSI_PERIOD    = 14                  # RSI 周期
KDJ_PERIOD    = 9                   # KDJ 周期
BOLL_PERIOD   = 20                  # 布林带周期
ATR_PERIOD    = 14                  # ATR 周期
```

### 相似度检索

```python
DTW_WEIGHT    = 0.5                 # DTW 权重
COSINE_WEIGHT = 0.5                 # 余弦相似度权重
```

**混合检索策略：**
1. 第一阶段：余弦相似度快速召回 top 100 候选
2. 第二阶段：DTW 精细排序（20% 窗口约束）
3. 第三阶段：加权融合得分（50% DTW + 50% 余弦）

## 相似度算法

### 余弦相似度（Cosine Similarity）

- **原理**：计算特征向量夹角的余弦值
- **优点**：计算快速（O(n)），适合大规模召回
- **缺点**：对时间错位敏感，要求严格对齐

### DTW（Dynamic Time Warping）

- **原理**：动态规划匹配时间序列，允许局部伸缩
- **优点**：捕捉形状相似性，容忍时间错位
- **缺点**：计算慢（O(n²)），需要窗口约束加速

### 混合检索

结合两者优势：
- 用余弦相似度快速过滤掉明显不相关的案例
- 用 DTW 在候选集中精细排序
- 加权融合得分，平衡速度和准确性

## 输出解读

### HTML 报告内容

1. **K 线图** - 当前股票最近60日走势 + MA5/MA20
2. **技术指标** - MACD 柱状图、RSI 指标
3. **归一化价格对比** - 当前 vs top 5 相似案例的价格走势
4. **收益分布直方图** - 相似案例的未来收益分布
5. **综合评分雷达图** - 胜率/盈亏比/收益/回撤控制四维评分
6. **分年度胜率** - 按年份统计胜率（检验时间稳定性）
7. **统计表格** - 胜率、平均收益、最大回撤、盈亏比等
8. **Top 5 相似案例** - 具体案例列表（股票代码、日期、相似度、后续收益）

### 关键指标

| 指标 | 含义 | 理想值 |
|------|------|--------|
| 胜率 | 相似案例中涨幅≥10%的比例 | ≥60% |
| 平均收益 | 相似案例的平均涨幅 | ≥15% |
| 盈亏比 | 平均盈利 / 平均亏损 | ≥3.0 |
| 最大回撤 | 相似案例的最大回撤 | ≤10% |
| 综合评分 | 0-100分，综合上述四项 | ≥70分 |

### 如何使用报告

**强信号（建议关注）：**
- 胜率 ≥ 60%
- 盈亏比 ≥ 3.0
- 综合评分 ≥ 70
- 分年度胜率稳定（不是某一年特别好）

**弱信号（谨慎对待）：**
- 胜率 < 50%
- 盈亏比 < 2.0
- 综合评分 < 50
- 分年度胜率波动大（可能过拟合）

**注意事项：**
- 历史相似 ≠ 未来必然重复
- 样本量太小（<10个）时统计不可靠
- 市场环境变化会影响有效性
- 结合基本面和其他技术分析综合判断

## 常见问题

### Q: 样本库应该包含哪些股票？

**A:** 建议策略：
- **行业代表股**：每个行业选2-3只龙头（覆盖不同行业特征）
- **市值分层**：大盘股 + 中盘股 + 小盘股（不同市值风格）
- **时间跨度**：至少3年数据（包含牛市/熊市/震荡市）
- **样本量**：50-100只股票，生成5000-10000个窗口样本

**不建议：**
- 只用自己持仓的股票（样本偏差）
- 只用某一行业（泛化能力差）
- 时间跨度太短（过拟合风险）

### Q: 为什么查询结果显示"无相似案例"？

**可能原因：**
1. 样本库太小或不包含相关行业
2. 当前形态过于特殊，历史上罕见
3. 成功案例定义太严格（10%涨幅 + 8%回撤）

**解决方案：**
- 扩大样本库（增加股票数量和时间跨度）
- 放宽成功案例标准（修改 config.py）
- 检查数据质量（是否有缺失值）

### Q: 胜率很高但实盘效果不好？

**可能原因：**
1. **过拟合** - 样本库时间跨度太短，只覆盖某种市场环境
2. **幸存者偏差** - 样本库只包含"活下来"的股票
3. **未来函数** - 特征计算中混入了未来信息（检查代码）
4. **市场环境变化** - 历史有效的形态在当前环境失效

**改进方法：**
- 分年度验证胜率稳定性
- 样本库包含退市股票
- 代码审查，确保无未来函数
- 结合宏观环境判断（牛市/熊市/震荡市）

### Q: 如何调整成功案例定义？

修改 `config.py`：

```python
# 更激进（追求高收益）
SUCCESS_RETURN_THRESHOLD = 0.20     # 20% 涨幅
SUCCESS_DRAWDOWN_LIMIT   = 0.12     # 12% 回撤

# 更保守（追求稳定）
SUCCESS_RETURN_THRESHOLD = 0.05     # 5% 涨幅
SUCCESS_DRAWDOWN_LIMIT   = 0.05     # 5% 回撤
```

**权衡：**
- 标准越严格，样本量越少，但质量越高
- 标准越宽松，样本量越多，但噪声越大
